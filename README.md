## 1. 프로젝트 개요
- 프로젝트 주제
    - 챗봇이 고객의 **문제해결 실패를 감지**하면, **콜센터 상담을 예약해**주고, **질문이력을 요약하여 상담사에게 제공**해주는 서비스

- 목표
  - 챗봇이 문제해결 실패를 빠르게 판단하도록 하여, 고객이 같은 내용을 여러번 설명하지 않도록 한다.
  - 적합한 상담부서 및 상담사를 자동 배정하도록 하여, 상담 스케쥴링을 간소화 및 최적화한다.
  - 고객이 챗봇과 나눈 대화를 미리 파악하고 상담을 시작할 수 있도록 하여, 불필요한 질문과 반복적인 안내를 최소화하여 상담 시간을 단축할 수 있도록 한다.

# 요구사항 정의서

## 1. 기능 요구사항
- [ ] **고객 감정 분석**: 고객의 감정을 실시간으로 분석하고, 이를 바탕으로 적절한 응대 또는 콜센터 이관
- [ ] **답변 어려움 판단**: 고객의 질문 내용, 이전 대화 이력 등을 분석하여 답변이 어려운 경우 콜센터로 이관
- [ ] **콜센터 연동**: 콜센터 시스템과 연동하여 고객 문의와 관련 정보를 상담사에게 전달, 이를 바탕으로 맞춤형 상담 제공
- [ ] **챗봇 성능 모니터링**: 챗봇 성능 지표를 실시간으로 모니터링하고, 데이터를 분석하여 챗봇 성능 개선

## 2. 비기능 요구사항
- [ ] **성능**: 빠른 응답(5초 이내), 감정 분석 정확도 80% 이상, 답변 어려움 판단률 90% 이상 확보
- [ ] **사용성**: 쉽고 편리한 인터페이스와 다양한 입력 방식을 지원, 명확하고 자연스러운 언어로 답변
- [ ] **신뢰성**: 챗봇 시스템의 높은 가용성, 예외 상황 발생에 대한 안정성
- [ ] **보안**: 고객의 개인정보를 안전하게 보호, 관련 법규 준수
- [ ] **확장성**: 향후 증가할 수 있는 사용자 수 및 데이터 양을 고려하여 확장 가능하도록 설계
- [ ] **유지보수**

# 프로젝트 설계서

## 1. 시스템 아키텍처
- **마이크로서비스 아키텍처**: 확장성 및 유지보수성 고려, 각 기능은 독립 서비스로 구성
- **주요 컴포넌트**:
  - 챗봇 인터페이스: React.js 기반의 사용자 인터페이스
  - 챗봇 처리 서비스: FastAPI/Flask 기반의 백엔드 서비스
  - 감정 분석 서비스: FastAPI/Flask 기반, Hugging Face Transformers 활용
  - 답변 어려움 판단 서비스: FastAPI/Flask 기반, 머신러닝 모델 활용
  - 콜센터 연동 서비스: FastAPI/Flask 기반, 콜센터 시스템 API 연동
  - 메시지 큐: Kafka 또는 RabbitMQ (비동기 처리)
  - 데이터베이스: MySQL, MongoDB

## 2. 데이터 설계
- **데이터베이스**
  - MySQL: 사용자 정보, 챗봇 시나리오 등 정형 데이터 저장
  - MongoDB: 대화 로그, 감정 분석 결과 등 비정형 데이터 저장
- **주요 엔티티**
  - 사용자, 대화, 메시지, 챗봇 시나리오, 콜센터 연동 로그
- **데이터 접근 및 보안**
  - ORM (SQLAlchemy) 및 MongoDB 드라이버를 사용하여 데이터에 접근
  - API Gateway를 통한 인증/인가 및 데이터 보안 적용

## 3. 기술 스택
- **데이터 수집**: Python, Selenium, RESTful API, Airflow
- **데이터 분석 및 모델링**: Pandas, NumPy, Scikit-learn
- **시각화**: Matplotlib, Seaborn, Plotly
- **클라우드 인프라**: AWS (EC2, RDS, S3)
- **컨테이너화/오케스트레이션**: Docker, Kubernetes
- **워크플로우 오케스트레이션**: Airflow
- **DevOps/배포**: GitHub, Jenkins, GitLab CI/CD
- **프론트엔드**: React.js
- **백엔드/API 서버**: Python (FastAPI, Flask), JWT
- **외부 API**: 엑사원 3.5 (2.4B), Llama 3 8B (Hugging Face Transformers)

## 4. 예상 문제 및 해결 방안
- **성능 문제**: 로드 밸런싱, 캐싱, 비동기 처리, DB 최적화
- **데이터 정확성 문제**: 데이터 품질 관리, 모델 개선, 오류 분석, 사용자 피드백 반영
- **시스템 연동 문제**: API 안정성 확보, 데이터 검증, 모니터링 강화, 롤백/복구 방안 마련
- **보안 문제**: 데이터 암호화, 접근 제어, 입력값 검증, 정기적 보안 점검
